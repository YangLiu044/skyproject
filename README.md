# 去中心化图片存证平台部署说明

## 📌 功能特点

这是一个**完全免费、无需服务器**的去中心化图片存证平台：

✅ **零成本** - 无需任何区块链费用，无需服务器  
✅ **即时使用** - 直接打开HTML文件即可使用  
✅ **唯一性保证** - 使用SHA-256算法生成64位唯一哈希  
✅ **本地存储** - 数据存储在浏览器本地，隐私安全  
✅ **永久验证** - 任何人都可以验证图片哈希

## 💡 使用说明

### 上传图片
1. 点击或拖拽图片到上传区域
2. 填写标题和描述（可选）
3. 点击"生成唯一哈希并存储"
4. 系统自动生成SHA-256哈希值
5. 图片保存在浏览器本地存储

### 浏览作品
- 点击"浏览作品"标签查看所有上传的图片
- 点击图片查看详细信息和完整哈希值
- 哈希值可用于验证图片真实性

### 验证哈希
任何人都可以通过以下方式验证：
1. 获得原始图片文件
2. 使用任何SHA-256工具计算哈希
3. 对比哈希值是否一致

## 🔧 技术原理

**SHA-256 哈希算法**：
- 生成64位十六进制唯一标识
- 碰撞概率：1/2^256（几乎不可能）
- 任何改动都会产生完全不同的哈希

**本地存储**：
- 使用浏览器 LocalStorage API
- 数据永久保存在用户设备
- 无需数据库或服务器

**IPFS CID模拟**：
- 生成类似IPFS的内容标识符
- 可升级集成真实IPFS网络

## 📊 存储限制

- 浏览器LocalStorage限制：5-10MB
- 建议图片压缩到1-2MB以下
- 可存储约50-100张图片

## 🔐 安全说明

1. **数据安全**：所有数据存储在本地，不会上传到任何服务器
2. **隐私保护**：完全离线运行，无需网络连接
3. **防篡改**：哈希值确保图片未被修改
4. **备份建议**：定期导出重要数据

## ❓ 常见问题

**Q: 数据会丢失吗？**  
A: 数据存储在浏览器中，清除浏览器数据会导致丢失。建议定期截图保存哈希值。

**Q: 可以多人使用吗？**  
A: 可以，但每个人的数据是独立的。如需共享，需要部署到网络服务器。

**Q: 支持哪些浏览器？**  
A: 支持所有现代浏览器（Chrome、Firefox、Safari、Edge）

**Q: 如何导出数据？**  
A: 可在浏览器控制台输入：`JSON.stringify(localStorage.getItem('images'))`
